# 第 0 课：先把 Agent 拆开

这一版故意不接真实大模型，也不直接上 LangGraph。目标是先看懂 Agent 的最小工程结构。

一个 Agent 至少包含四块：

1. 指令：它要解决什么问题。
2. 工具：它能查询哪些外部信息。
3. 状态：它已经查到了什么证据。
4. 决策循环：下一步查什么，最后怎么回答。

本项目第一版对应关系：

- `data/`：模拟企业结构化数据。
- `knowledge_base/`：模拟企业知识库。
- `tools.py`：Agent 可调用的工具。
- `agent.py`：决定调用哪些工具，并把证据整理成答案。
- `main.py`：命令行入口。
- `tests/`：确认 Agent 能找到预期根因。

当前我们把“模型决策”写成了确定性规则。下一步会把其中一部分替换成真实 LLM 调用，再往后用 LangGraph 管理流程状态。
